## Overview

This project implements a full **SPI (Serial Peripheral Interface)** communication system in **SystemVerilog**, including a master controller, a mock slave, and a complete sensor acquisition pipeline. The design integrates SPI communication with a beamforming application, where sensor data is read via SPI and processed through a ROM-based phase lookup and memory management system.

---

## SPI Protocol Background

SPI is a synchronous, full-duplex serial communication protocol using four signals:

| Signal | Direction | Description |
|--------|-----------|-------------|
| `SCLK` | Master → Slave | Serial clock generated by the master |
| `MOSI` | Master → Slave | Master Out Slave In — data from master |
| `MISO` | Slave → Master | Master In Slave Out — data from slave |
| `CS_n` | Master → Slave | Chip Select (active-low) — selects the target slave |

The four SPI modes (Mode 0–3) are defined by clock polarity (CPOL) and clock phase (CPHA).

## Design Highlights

- **SPI Master:** Fully parameterised for data width and clock divider. Handles CS assertion, serial shifting, and MISO sampling on the correct clock edge.
- **Mock Slave/Sensor:** Simulates a sensor device that responds to SPI transactions with predefined phase-coded data.
- **Beamforming Pipeline:** Integrates the SPI acquisition with a phase ROM lookup and a memory manager, forming a simple digital beamforming front-end.
- **Testbenches:** Multiple layered testbenches verify individual modules and the full integration path.
